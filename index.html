Tre olika sätt att publicera

I uppgiften ingick att jämföra tre olika sätt att publiera material på webben. Jag har valt att jämföra de tre plattformarna, Netlify, Github Pages och One.com.

Netlify Github Pages One.com

  • Netlify

    Netlify är ett free hosting-alternativ som tillåter dig att publicera statiska sidor gratis. För att publicera din webbsida kopplar du ditt Github-repository (alternativt GitLab eller Bitbucket). Det är ett snabbt och enkelt sätt att få upp din sida via Netlify. Denna plattform är dock inte att rekommendera om du vill publicera större webbplatser, då det finns begränsningar i utrymmet.

  • Github Pages

    Att publicera sin webbsida via Github Pages är enkelt. Det enda som behövs är ett Github-konto. Fördelarna med Github pages är att det är gratis, och att din webbsida uppdateras automatiskt när du uppdaterar ditt Github-repository. Det är även fritt från reklam. Det är främst till för att hosta statiska sidor, men det går även att publicera dynamiska sidor, även om det kräver lite mer arbete.

  • One.com

    Är ett webbhotell som, precis som de flesta andra webbhotell, erbjuder paketlösningar efter vad som passar din webbplats bäst. Man delar webbserver med andra och du behöver därför inte oroa dig för att behöva underhålla och konfigurera servern, utan det sköts av One.com. Det är ett jämförelsevis billigt webbhotell, som erbjuder paket där bland annat obegränsade e-postkonton, databas och SSL-certifikat ingår. De hjälper också till med domän-hantering.
